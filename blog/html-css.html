Class and ID selectors

A crash course in best practices

23 June, 2023

Introduction

When we style code, we need to let our stylesheet know which elements of our webpage we want it to style. That's where class and ID selectors come in.

Generally speaking, class and ID selectors are the names we give to specific elements in our HTML code. For example, we can create a div container and give it the name pink-div. When we then go into our CSS file and write some style rules for pink-div (such as giving it a padding of 5px, making the background colour pink and giving it a fuschia border), those style rules will only be applied to any element which is named pink-div. This means we can style specific elements in a lot of different ways, creating visually unique sections and pages on a website.

In this blog post, we'll be exploring the key differences between class and ID selectors, and then we'll delve into the best practices for each of these.

What is a class selector?

If you want to apply the exact same set of style rules to lots of different HTML elements, then using the class selector is the right choice. The class selector represents a 'class' of elements which all share the same styles, and it's handy because you only need to define those styles once in your CSS code to apply them multiple times in your HTML code.

Let's say that you want to style a main menu. Each item in your menu ('Home', 'About', etc) has the exact same name: "menu-item". This is because we want each of our menu items to look exactly the same on our website. 

          
            <ul>
            <li class="menu-item"><a href="#">Home</a></li>
            <li class="menu-item"><a href="#">About</a></li>
            <li class="menu-item"><a href="#">Articles</a></li>
            <li class="menu-item"><a href="#">Item</a></li>
            </ul>

Now, we can go into our CSS stylesheet and create a set of rules for our menu items. 

          
            .menu-item {
              padding: 8px;
              width: 100%;
              color: pink;
            }

Once we save that and refresh, all of the items with the class 'menu-item' now look identical. That's the magic of using a class selector!

What is an ID selector?

Let's say that we want to style the overall look of the main menu itself. Because the main menu is going to be the only element which will have this style applied to it, we don't need to ensure this style can be used anywhere else on the page. That's where the ID selector comes in.

The ID selector works in a very similar way to the class selector, but there's a key difference. You can only use an ID to style a single element in your HTML code.

Our overall main menu has an ID called main-menu, and we are only going to use this name once in our HTML code. 

          
            <div id="main-menu">
              <ul>
                  <li class="menu-item"><a href="#">Home</a></li>
                  <li class="menu-item"><a href="#">About</a></li>
                  <li class="menu-item"><a href="#">Articles</a></li>
                  <li class="menu-item"><a href="#">Item</a></li>
                </ul>
            </div>

Now, let's go into our CSS stylesheet again. We can create a set of rules for main-menu as well, but notice the difference in syntax. By using a '#' instead of a '.', we're telling the stylesheet that this particular set of style rules is an ID selector and will therefore only be used once in our HTML code. 

          
        #main-menu {
          background-color: white;
          border: solid black 1px;
          margin: 2px;
          padding: 2px;
        }

When we save this and refresh our webpage, we now have our style applied to only the main menu element on our webpage.

The class and ID selectors essentially function in the same way, but it's how you use them that counts.

Best practices

You should always reserve ID selectors for styling a single element in your HTML code. If you know you'll only be using a specific element once (such as the main menu from our earlier example), then go ahead and use an ID selector for this.

Class selectors are best to use when you have lots of elements in your HTML code which will need to share the same style (such as our menu items from our earlier example).

Ensuring that you use each of these selectors in the correct situations will make your code look cleaner and run better, and will be easier for other developers to read.

Here's an example of what not to do:

          
            
            <head>
            <style>
              #pink-1 {
                color: pink;
                font-size: 10px;
                font-family: sans-serif;
              }

              #pink-2 {
                color: pink;
                font-size: 10px;
                font-family: sans-serif;
              }

              #pink-3 {
                color: pink;
                font-size: 10px;
                font-family: sans-serif;
              }
            </style>
          </head>
            <html>
              <body>
                <p id="pink-1">This is a pink paragraph.</p>

                <p id="pink-2">This is another pink paragraph.</p>

                <p id="pink-3">And another.</p>
              </body>
            </html>

Versus how this code can be cleaned up with best practice applied: 

          
            <head>
            <style>
              .pink-paragraph {
                color: pink;
                font-size: 10px;
                font-family: sans-serif;
              }
            </style>
          </head>
            <html>
              <body>
                <p class="pink-paragraph">This is a pink paragraph.</p>
  
                <p class="pink-paragraph">This is another pink paragraph.</p>
  
                <p class="pink-paragraph">And another.</p>
              </body>
            </html>

It's also important to think about the order of operations when you write your CSS code. When your browser interprets CSS, it looks over all the selectors and determines the order of operations it will 'cascade' in. ID selectors always take priority over class selectors in CSS code.

To sum up

ID selectors are a great choice if you need to style a single element which isn't repeated in your webpage. For styling multiple elements with the same set of style rules, stick to class selectors. Your code will look and work better, and other developers will thank you for it.

Happy coding!
